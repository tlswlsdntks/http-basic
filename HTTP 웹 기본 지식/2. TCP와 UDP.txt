IP 와 TCP 패킷 정보)
IP: 출발지 IP, 목적지 IP, ... (전송데이터 포함)
TCP: 출발지 PORT, 목적지 PORT, 전송제어, 순서, 검증정보, ... (전송데이터 포함)

TCP
: 전송 제어 프로토콜

특징)
1) 연결지향(TCP 3 way handshake, 가상연결) -> SYN(접속 요청), ACK(요청 수락)
 1.SYN,
 2.SYN + ACK,
 3.ACK (3.ACK와 함께 데이터 전송도 가능)

2) 데이터 전달 보증: 데이터 전송 -> 데이터 전송 완료에 대한 답변을 재전송
3) 순서보장: 패킷1, 패킷2, 패킷3 -> 패킷1, 패킷3(X), 패킷2 -> 패킷2부터 다시 보내라고 요청
4) 신뢰할 수 있는 프로토콜
5) 현재 대부분 TCP 사용

UDP
:사용자 데이터 프로토콜

특징)
1) 하얀 도화지(기능이 거의 없음)
2) 연결지향(TCP 3 way handshake, 가상연결) -> X
3) 데이터 전달 보증 -> X
4) 순서보장 -> X
5) 데이터 전달 및 순서가 보장되지 않지만, 단순하고 빠름
6) PORT 및 체크섬(네트워크를 통해서 전송된 데이터의 값이 변경되었는지(무결성)를 검사하는 값)정도의 기능만 추가
